Here are some safety plans not to bankrupt myself:

To host this safely without bankrupting yourself or crashing browsers, apply these three layers of defense:

Layer 1: Gzip Compression (The "Free Lunch")

Statevector JSON is highly compressible (text-based numbers). Enabling Gzip on your server (e.g., in FastAPI/Uvicorn or Nginx) can reduce the 16MB payload to ~2-3 MB.

    Benefit: Reduces billing by ~80% and keeps you under the AWS API Gateway 10MB limit.

    How: Add GZipMiddleware to your FastAPI app.

Python

from fastapi.middleware.gzip import GZipMiddleware
app.add_middleware(GZipMiddleware, minimum_size=1000)

Layer 2: The "Presigned URL" Pattern (For Downloads)

For your "Download Full Report" feature, do not stream the data through your API server.

    Save to Object Storage: When the benchmark finishes, save the full JSON to AWS S3 (or Google Cloud Storage).

    Generate a Link: The API returns a Presigned URL (a temporary, secure link to the file on S3).

    Direct Download: The user's browser downloads the file directly from S3.

    Benefit: This unloads your API server's CPU/RAM and is often cheaper or has a better free tier for pure storage retrieval.

Layer 3: Hard Cap on "Free" Tier

In your docker-compose.yml (or cloud env), set a hard limit on the n_shots or qubits for unauthenticated users.

    Rule: MAX_QUBITS_PUBLIC=14 (Safe for browser/bandwidth).

    Upsell/Auth: If qubits > 14, require an API key or login. This prevents anonymous abuse of your expensive 20-qubit simulations.
